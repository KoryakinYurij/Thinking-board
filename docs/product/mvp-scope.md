# MVP Scope

## Product Framing

This product is a personal or small-team `AI-assisted task execution app`.

The MVP should feel optimized for:
- quickly capturing rough ideas
- using AI to expand messy intent into clearer work
- using AI to decompose a task into executable steps
- reviewing AI output before it mutates committed data
- executing accepted work through list, focus, and kanban surfaces

The MVP should not feel optimized for:
- generic chat sessions without task outcomes
- project portfolio management
- advanced permissions
- workflow engines generated by AI
- highly customizable enterprise boards

## MVP In

- create and edit a `capture item`
- promote a capture item into a committed task
- create and edit a task manually
- complete and reopen a task
- archive and delete a task
- run `Expand with AI` on a capture item or task
- run `Decompose with AI` on a task
- review AI output before acceptance
- accept all or part of an AI suggestion set
- create subtasks from accepted decomposition output
- view committed tasks in list, focus, or kanban surfaces
- move committed top-level tasks within and between board columns
- persist capture, task, subtask, and acceptance state
- basic filtering such as text query, status, or focus state
- usable mobile and desktop layout

## MVP Out

- autonomous AI that edits canonical data without review
- realtime collaboration
- advanced scheduling automation
- knowledge graph features beyond task usefulness
- comments and activity feed
- complex assignee permissions
- portfolio analytics and reporting
- arbitrary user-defined workflow engines

## UX Expectations

- capture must be faster than organization
- expansion and decomposition must be distinct user actions
- AI output must be understandable, reviewable, and partially acceptable
- task completion must still be easier than dragging a card
- board movement must not be the only way to change execution status
- important actions must work on mobile without hover
- board layout must remain understandable with long task titles
- unaccepted AI output must not look like committed work

## Delivery Order

1. establish terminology and domain model
2. ship capture and triage
3. ship expansion flow
4. ship decomposition flow
5. ship accepted-task execution surfaces
6. ship persistence and failure handling
7. harden mobile UX, accessibility, and eval quality
